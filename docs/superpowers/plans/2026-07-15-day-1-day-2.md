# TutorLab Day 1–2 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: use native subagent-driven development task-by-task. Each task is implemented by a fresh native subagent, reviewed by the orchestrator, validated with the full project gate, committed separately, and annotated with a git note.

**Goal:** Deliver the approved Day 1–2 milestone from project creation through budgeted multi-document analysis and editable compact course-model synthesis.

**Architecture:** One Next.js 15 App Router application owns UI and route handlers. PostgreSQL/Prisma persists project access, source processing, document analyses, jobs, and immutable course-model versions. OpenAI integrations are isolated behind server-only adapters; canonical Zod schemas define every boundary.

**Tech Stack:** Next.js 15, React, TypeScript, Tailwind CSS, shadcn/ui, Prisma, PostgreSQL, Zod, OpenAI JavaScript SDK, Vitest, Testing Library, Playwright, Docker Compose.

## Task contract

### Task 1: Scaffold the application and local runtime

**Files:** foundation/configuration files and baseline test configuration.

**Consumes:** approved SPEC and design only.

**Produces:** buildable Next.js application; Docker PostgreSQL; validated `env`; singleton `db`; server-only `getOpenAIClient(): OpenAI`; lint/typecheck/test/build scripts.

**Acceptance:** clean install succeeds; database service is reachable; baseline lint, typecheck, tests, and production build pass; secrets remain ignored.

**Commit:** `chore: scaffold tutorlab application`

### Task 2: Define canonical schemas and deterministic fixtures

**Files:** `lib/schemas/**`, JSON fixtures, schema unit tests.

**Consumes:** approved artifact contracts and budget defaults.

**Produces:** inferred TypeScript types and Zod schemas for `TeachingBrief`, `SourceDocument`, `DocumentAnalysis`, `CourseModel`, `CourseModelPatch`, and `PipelineJob`; schema-versioned deterministic fixtures.

**Interfaces:** `parseTeachingBrief(input)`, `parseSourceDocument(input)`, `parseDocumentAnalysis(input)`, `parseCourseModel(input)`, `parseCourseModelPatch(input)`, `parsePipelineJob(input)` return canonical typed values or structured validation errors.

**Acceptance:** valid fixtures parse; malformed evidence, permissions, coverage, and oversized compact-model fixtures fail deterministically.

**Commit:** `feat: define milestone data contracts`

### Task 3: Add persistence, project authorization, and project APIs

**Files:** Prisma schema/migrations, auth token, project repositories/access, project routes, integration tests.

**Consumes:** Task 2 schemas; environment/database clients.

**Produces:** persisted project lifecycle and immutable artifact relations; HTTP-only edit session; authorized create/read/mutate boundary.

**Interfaces:** `createProject(input): Promise<{project; editToken}>`; `requireProjectAccess(request, projectId): Promise<Project>`; `saveTeachingBrief(projectId, patch): Promise<Project>`.

**Acceptance:** token material is never stored plaintext; unauthorized/cross-project mutations fail; refresh preserves stage and brief.

**Commit:** `feat: add project persistence and edit access`

## Build order and delegation

Tasks execute sequentially in numeric order because this is a greenfield repository and each slice consumes contracts or persistence landed by earlier tasks. Every task is delegated to a fresh native implementation subagent with no git authority. The orchestrator reviews the diff, normalizes files, runs the full project gate, stages only task files, commits immediately, adds the required git note, and then marks the TODO sub-item complete with its commit hash.

## Risks and controls

- **Greenfield bootstrap drift:** pin the approved major stack and validate build immediately in Task 1.
- **Schema mismatch across workers:** Task 2 owns all canonical names and versions; later tasks import rather than redefine them.
- **Database-dependent test friction:** Docker Compose and isolated test database conventions land before persistence work.
- **Provider cost and nondeterminism:** all automated tests mock the OpenAI boundary; fixture mode is explicit; live calls are reserved for the curated flow.
- **Large-workspace context overflow:** per-document analysis, content-hash reuse, limited concurrency, category reduction, and compact synthesis are mandatory.
- **Protected-content leakage:** source permissions are enforced at ingestion, synthesis selection, evidence rendering, and future retrieval boundaries.
- **Missing owner-supplied binaries:** implementation and deterministic tests proceed; the live milestone gate remains pending until the three files are present.
- **Cross-task regressions:** the orchestrator runs the complete suite, typecheck, lint, and relevant build/browser checks after every delegated task.
