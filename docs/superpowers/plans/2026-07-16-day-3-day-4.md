# Day 3–4 Tutor Design and Evaluation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Deliver a teacher-selected, compiled, course-grounded tutor and six-scenario persisted evaluation loop.

**Architecture:** Extend the existing project-scoped persistence, fixture runtime, provider adapters, job polling, and stage pages. New schema-bearing artifacts are validated at every boundary and immutable once versioned. Live adapters use server-only OpenAI routes; fixture adapters implement the same contracts for deterministic unit, integration, and browser coverage.

**Tech Stack:** Next.js App Router, TypeScript, Prisma/PostgreSQL, Zod, OpenAI Responses API, Vitest, Playwright.

## Risks and controls

- **Provider variability:** fixtures share contracts with live adapters; the fixed attack sequence controls only learner input, never tutor output or result judgment.
- **Protected answer exposure:** server-side retrieval filters and deterministic leakage checks are independent safeguards.
- **Long-running work:** each stage/job/run persists after discrete activity and refreshes by polling.
- **Schema drift:** all artifacts use strict versioned Zod schemas at adapter, service, persistence, and route boundaries.
- **Costs/rate limits:** scenario execution is capped and concurrency is fixed at three.
